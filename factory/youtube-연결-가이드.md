# 유튜브 자동 업로드 연결 가이드 (사장님용)

> 목표: 유튜브가 "이 앱이 내 채널에 영상 올려도 돼"라고 허락하는 **허가증 3개**를 만든다.
> 1·2번은 사장님이, 3번(리프레시 토큰)은 AI가 도와줍니다.
> ※ 유튜브 채널을 만든 **그 구글 계정**으로 로그인한 상태에서 진행하세요.

---

## 1단계 — YouTube Data API v3 켜기

1. 아래 링크 열기 (TTS 켤 때 쓴 그 프로젝트가 맞는지 상단에서 확인)
   👉 https://console.cloud.google.com/apis/library/youtube.googleapis.com
2. 파란 **[사용]**(Enable) 버튼 클릭
3. 끝. (이미 켜져 있으면 "관리"로 보이는데 그대로 두면 됩니다)

---

## 2단계 — OAuth 동의 화면 만들기

"누가 허락하는지" 앱 정보를 등록하는 단계입니다.

1. 링크 열기 👉 https://console.cloud.google.com/apis/credentials/consent
2. User Type(사용자 유형): **외부(External)** 선택 → **[만들기]**
3. 앱 정보 입력:
   - 앱 이름: `오즈백 릴스 업로더` (아무거나)
   - 사용자 지원 이메일: 본인 이메일 선택
   - 개발자 연락처 이메일: 본인 이메일 입력
   - → **[저장 후 계속]**
4. 범위(Scopes) 화면: 아무것도 안 하고 **[저장 후 계속]**
5. 테스트 사용자(Test users) 화면: **[+ ADD USERS]** → **본인 구글 이메일 추가** → **[저장 후 계속]**
6. 요약 화면 → **[대시보드로 돌아가기]**

### ⭐ 중요 — "게시" 눌러주기 (안 하면 일주일마다 끊깁니다)
7. 같은 동의 화면 페이지에서 **[앱 게시]**(PUBLISH APP) 버튼 클릭 → 확인
   - "인증 필요" 안내가 떠도 **개인 채널 용도라 그냥 두면 됩니다** (나중에 로그인 때 "확인되지 않은 앱" 경고가 나오는데, 본인 채널이니 "계속"으로 통과합니다)
   - 이걸 해야 허가증(리프레시 토큰)이 **안 끊기고 계속** 유지됩니다

---

## 3단계 — OAuth 클라이언트 ID 만들기 (여기서 나오는 2개를 저한테 주세요)

1. 링크 열기 👉 https://console.cloud.google.com/apis/credentials
2. 상단 **[+ 사용자 인증 정보 만들기]** → **[OAuth 클라이언트 ID]**
3. 애플리케이션 유형: **데스크톱 앱(Desktop app)** 선택
4. 이름: `오즈백 업로더` (아무거나) → **[만들기]**
5. 팝업에 나오는 **클라이언트 ID**와 **클라이언트 보안 비밀(Secret)** 두 개를 복사
   (또는 **[JSON 다운로드]** 눌러 파일로 받아도 됩니다)

### 저한테 주실 것
- **클라이언트 ID** (예: `1234...apps.googleusercontent.com`)
- **클라이언트 보안 비밀(Secret)** (예: `GOCSPX-...`)

→ 이 두 개를 채팅에 붙여넣어 주시면, 제가 **리프레시 토큰 받는 도구**를 만들어 드립니다.
   사장님은 구글 로그인 한 번 클릭만 하시면 유튜브 자동 게시가 완성됩니다.

> 🔒 이 값들은 .env / GitHub Secrets 에만 안전하게 보관하고 외부 유출하지 않습니다.
